# TCP

Вкладка TCP — основная часть настройки обхода DPI. Содержит три подвкладки: **Общее**, **Фрагментация** и **Faking**.

## Общее

### Лимит TCP пакетов соединения

Сколько байт соединения анализировать. После этого лимита пакеты проходят без модификации. Увеличение значения замедляет обработку, уменьшение может привести к тому, что b4 не успеет модифицировать нужные пакеты.

### Задержка между пакетами

Диапазон задержки (мс) между отправкой фрагментов. Некоторые DPI-системы собирают фрагменты по таймеру — задержка может помочь обойти эту логику.

### Фильтр портов

Порты, на которых работает обход. Формат iptables: `443` или `443,80`. Если не указан — обрабатываются все порты.

### Отбросить SACK

Удаляет опцию SACK из TCP-пакетов. Некоторые DPI-системы используют SACK для сборки фрагментов.

### Дублирование пакетов

Отправляет каждый пакет несколько раз. Полезно, если провайдер отбрасывает часть пакетов.

## Фрагментация

Основной инструмент обхода DPI. Суть: разбить TCP-пакет на части так, чтобы DPI не смог собрать их и прочитать SNI.

### Метод фрагментации

| Метод | Описание |
| --- | --- |
| **tcp** | TCP-фрагментация — разделение на уровне TCP-сегментов |
| **ip** | IP-фрагментация — разделение на уровне IP-пакетов |
| **tls** | Разделение TLS-записи на несколько записей |
| **oob** | Out-of-Band — отправка данных через urgentpointer |
| **combo** | Комбинация нескольких методов с приманками и перемешиванием |
| **hybrid** | Гибрид combo и disorder |
| **disorder** | Отправка фрагментов в изменённом порядке с jitter |
| **extsplit** | Разделение по границе TLS-расширений |
| **firstbyte** | Отделение первого байта от остальных данных |
| **none** | Без фрагментации |

### Пул стратегий

Если включён пул, b4 случайным образом выбирает стратегию из пула для каждого соединения. Это затрудняет DPI-системе адаптацию к конкретному методу.

### Обратный порядок

Отправляет фрагменты в обратном порядке (последний фрагмент первым).

### Настройки по методам

#### TCP/IP фрагментация

- **Умный раскол SNI** — автоматически находит середину SNI-поля и разделяет там
- **Фиксированная позиция раскола** — ручная настройка точки разделения (смещение в байтах)

#### Combo

- **Приманка (Decoy)** — отправляет ложный пакет перед настоящим
- **First Byte / Extension / SNI Split** — точки разделения
- **Режим перемешивания** — middle, full, reverse
- **Fake Per Segment** — отправка фейковых пакетов между фрагментами

#### Disorder

- **SNI Split** — разделение по SNI
- **Режим перемешивания** — full, reverse
- **Jitter** — случайная задержка между фрагментами (мин/макс, мкс)
- **Шаблон перекрытия** — данные для fake-перекрытия: tls12, tls11, http_get, zeros, custom

#### OOB

- **Позиция OOB** — смещение urgent-указателя
- **OOB байт** — байт, отправляемый через OOB

#### TLS Record

- **Позиция раскола TLS записи** — где разделить TLS-запись

## Faking

Методы отправки ложных пакетов для обмана DPI.

### SNI Faking

Отправляет пакеты с поддельным SNI до настоящих данных. DPI видит фейковый SNI, принимает решение на его основе, а настоящие данные проходят незамеченными.

| Параметр | Описание |
| --- | --- |
| **Стратегия** | ttl — пакет с малым TTL (не дойдёт до сервера); randseq — случайный seq; pastseq — прошедший seq; tcp_check — плохая контрольная сумма; md5sum — TCP MD5; timestamp — устаревший timestamp |
| **Тип пэйлоада** | Random, Chrome ClientHello, DuckDuckGo ClientHello, файл, нули, инвертированный |
| **TTL** | Time-to-live для фейковых пакетов (1–64). Подбирается так, чтобы пакет дошёл до DPI, но не до сервера |
| **Количество пакетов** | Сколько фейковых пакетов отправить (1–20) |

### SYN Fake

Отправляет фейковые SYN-пакеты перед реальным соединением.

- **MD5** — добавляет TCP MD5-опцию
- **Длина пэйлоада** — размер данных в фейковом SYN
- **TTL** — TTL для фейковых SYN

### Десинхронизация (Desync)

Отправляет пакеты, которые рассинхронизируют состояние DPI-сессии:

| Режим | Описание |
| --- | --- |
| **rst** | Отправка RST — DPI считает соединение закрытым |
| **fin** | Отправка FIN — DPI считает соединение завершённым |
| **ack** | Отправка ACK с неверными данными |
| **combo** | Комбинация нескольких методов |
| **full** | Все методы одновременно |

### Манипуляция TCP-окном

Изменение значения TCP Window для влияния на поведение DPI:

| Режим | Описание |
| --- | --- |
| **zero** | Нулевое окно — заставляет отправителя приостановить передачу |
| **random** | Случайные значения окна |
| **oscillate** | Чередование заданных значений |
| **escalate** | Постепенное увеличение |

### Incoming (обход входящего трафика)

Обработка ответов сервера, а не запросов. Используется для обхода DPI, анализирующего ответы:

- **fake** — фейковые ответы
- **reset** — RST-пакеты
- **fin** — FIN-пакеты
- **desync** — десинхронизация
- **Порог** (мин/макс КБ) — применять только к ответам определённого размера

### Мутация ClientHello

Изменение структуры TLS ClientHello для обхода сигнатурных проверок:

| Режим | Описание |
| --- | --- |
| **grease** | Добавление GREASE-расширений |
| **padding** | Добавление padding до заданного размера |
| **fakeext** | Добавление поддельных TLS-расширений |
| **fakesni** | Добавление поддельных SNI-записей |
| **random** | Случайная комбинация методов |
| **advanced** | Ручная настройка всех параметров |
