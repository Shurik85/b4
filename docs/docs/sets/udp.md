# UDP

Вкладка UDP управляет обработкой UDP-трафика. Основные задачи: блокировка или подмена QUIC-трафика и обработка STUN.

## Фильтр QUIC

QUIC — протокол, который используют браузеры вместо TCP (YouTube, Google и другие). Некоторые провайдеры блокируют QUIC, заставляя браузер переключиться на TCP, где b4 может применить обход.

| Режим | Описание |
| --- | --- |
| **disabled** | Не фильтровать QUIC |
| **all** | Блокировать все QUIC-пакеты — браузеры переключатся на TCP |
| **parse** | Проверять домен в QUIC-пакете и блокировать только те, что совпадают с целями сета |

Режим **all** — простой и надёжный. Режим **parse** — точнее, но требует, чтобы домены были настроены в целях сета.

## Фильтр портов

Какие UDP-порты обрабатывать. Формат: `443,80,53`. Если не указан — все порты.

## Фильтровать STUN

Блокирует STUN-пакеты. STUN используется для WebRTC (видеозвонки, голосовые чаты). Блокировка STUN может помочь с обходом определённых видов фильтрации.

## Режим действия

Если трафик подходит под фильтры, к нему применяется одно из действий:

| Режим | Описание |
| --- | --- |
| **drop** | Отбросить пакет |
| **fake** | Отправить поддельный ответ |

## Настройки Fake-режима

При режиме **fake** доступны дополнительные параметры:

- **Стратегия подделки** — none, ttl, checksum
- **Количество подделок** — сколько фейковых пакетов отправить (1–20)
- **Размер подделок** — размер фейкового пакета (32–1500 байт)
- **Задержка между пакетами** — задержка отправки (0–1000 мс)
