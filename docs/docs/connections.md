# Соединения

Раздел отображает TCP/UDP-соединения, проходящие через b4, в реальном времени.

<!-- screenshot: таблица соединений -->

## Таблица

| Столбец | Описание |
| --- | --- |
| **Время** | Время соединения (ЧЧ:ММ:СС) |
| **Протокол** | TCP, UDP, P-TCP, P-UDP (P = прокси) |
| **Сет** | Название сета, если домен или IP совпал с настроенными целями. Пустое — соединение не обрабатывается |
| **Домен** | SNI-домен из TLS-рукопожатия. Рядом может отображаться версия TLS |
| **Источник** | IP отправителя. Если устройство опознано — показывает имя/vendor |
| **Назначение** | IP:порт получателя. Можно определить ASN через кнопку с иконкой сети |

## Добавление доменов

Если домен не входит ни в один сет, рядом с ним отображается иконка «+». При нажатии открывается диалог:

1. Выбор шаблона домена — от точного совпадения до широкого (все поддомены)
2. Выбор сета, в который добавить домен
3. Нажмите **Добавить домен**

<!-- screenshot: диалог добавления домена из таблицы соединений -->

## Добавление IP

Клик по IP назначения открывает диалог с возможностями:

- **Загрузить сетевую информацию** — получить данные об IP через RIPE
- **Обогатить через IPInfo** — получить организацию, hostname, расположение (нужен токен в настройках)
- Добавить один IP или все префиксы ASN в сет
- Выбрать CIDR-маску (/32, /24, /16 и т.д.)

## Фильтрация

Поле фильтра поддерживает комбинирование условий:

- `+` — объединение (AND)
- `!` — исключение (NOT)
- Фильтр по полям: `protocol:tcp`, `domain:youtube.com`, `asn:AS13335`, `device:имя`

Примеры:

- `tcp+!domain:google.com` — TCP-соединения, кроме Google
- `protocol:udp` — только UDP
- `domain:youtube` — соединения с youtube в домене

## Режим отображения

Переключатель **Только домены / Все пакеты**:

- **Только домены** (по умолчанию) — показывает только соединения с распознанным SNI
- **Все пакеты** — показывает весь трафик, включая пакеты без домена

## Горячие клавиши

| --- | --- |
